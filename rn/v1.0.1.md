# cyncia 1.0.1

Follow-up release on top of [v1.0.0](v1.0.0.md). Adds an opt-in Claude Code
emission mode for rules, a project-level `cyncia.conf` managed by the
installer, and a handful of doc/cleanup fixes. Default behavior is unchanged
from v1.0.0.

## Highlights

- **`claude_rules_mode` config for Claude Code rules.** A new
  `.cyncia/cyncia.conf` key controls how `rules/<name>.md` is emitted for
  Claude Code:
  - `claude-md` *(default, same as v1.0.0)* — rule bodies are merged into
    `CLAUDE.md` by `sync-agent-guidelines`.
  - `rule-files` — each rule is written to `.claude/rules/<name>.md` by
    `sync-rules`, and `CLAUDE.md` imports them via Claude `@`-memory imports
    so they load with the same priority as `CLAUDE.md` itself.
- **Schema-managed `cyncia.conf`.** The installer carries a schema for
  `cyncia.conf` and:
  - creates the file from the schema when missing,
  - prompts (default **yes**) to add properties newly introduced in this
    cyncia version,
  - prompts (default **no**) to remove properties no longer in the schema.
  Existing configs are otherwise left alone.
- **Docs.** README/`cyncia.md` gain a Configuration section documenting
  `cyncia.conf` and the `claude_rules_mode` values.

## Fixes / cleanups

- Fix `Cyncya` → `Cyncia` typo in a PowerShell helper name and variable.
- Drop a redundant import-explanation paragraph that `sync-agent-guidelines`
  was emitting into `CLAUDE.md` in `rule-files` mode.
- Portable `sed` usage in installer; minor doc fixes.

## Tests

- 11 new bats tests covering `cyncia.conf` create / preserve / add-missing /
  remove-unsupported flows and both `claude_rules_mode` modes (including
  `--clean` of stale per-rule files and invalid-mode fallback).
- Pester coverage extended for the PowerShell sync path of the new mode.

Total: 76 bats tests + Pester suite.

## Upgrading from 1.0.0

No action required. Without `cyncia.conf`, or with
`claude_rules_mode: claude-md`, behavior matches v1.0.0 exactly. To opt in
to per-rule files for Claude Code, re-run the installer (it will offer to
add the `claude_rules_mode` key) and set:

```yaml
claude_rules_mode: rule-files
```

then re-run `sync-all`.

## Installation

```bash
curl -fsSL https://raw.githubusercontent.com/crestreach/cyncia/main/install/install.sh | bash
```

To pin this release:

```bash
curl -fsSL https://raw.githubusercontent.com/crestreach/cyncia/main/install/install.sh \
  | bash -s -- --ref v1.0.1
```
